# AMO-docs

Проект аналога виджета "Документы" для amoCRM.

## Этапы реализации

### Этап 1. Базовая платформа (текущий)
- Каркас backend API.
- Готовность к деплою на Render.
- Базовая структура для будущего виджета amoCRM.

### Этап 2. Интеграция amoCRM
- OAuth/авторизация интеграции.
- Чтение данных сделки, контактов, компании, товаров.
- Базовый UI виджета в карточке сделки.

### Этап 3. Генерация документов
- Шаблоны и переменные.
- Генерация документов по API.
- Хранение документов и отображение в сделке.

### Этап 4. Автоматизация
- Генерация по событиям этапов воронки.
- Счетчики документов.
- Ссылки на просмотр, редактирование, PDF.

### Этап 5. Расширенные функции
- Сложная табличная часть товаров/услуг.
- Форматирование (пропись, даты, условия, функции).
- QR-коды, архив документов, интеграции с внешними сервисами.

## Текущая структура

- `backend/` — API и бизнес-логика.
- `widget/` — фронтенд виджета amoCRM (будет развиваться по этапам).
- `render.yaml` — инфраструктура деплоя на Render.

## Запуск backend локально

1. Перейти в `backend/`.
2. Установить зависимости: `npm install`.
3. Создать `.env` по примеру `.env.example`.
4. Запустить dev-сервер: `npm run dev`.

Проверка:
- `GET /health` — состояние сервиса.
- `GET /api/v1/documents` — тестовый список документов.

## Этап 2: amoCRM OAuth и чтение сделки

Добавлены endpoints:

- `GET /api/v1/amocrm/status` — текущий статус подключения.
- `GET /api/v1/amocrm/oauth/url` — получить ссылку на авторизацию в amoCRM.
- `POST /api/v1/amocrm/oauth/exchange` — обменять `code` на токены.
- `POST /api/v1/amocrm/oauth/refresh` — обновить access token.
- `GET /api/v1/amocrm/leads/:leadId` — получить данные сделки (с контактами/компанией).

Пример обмена кода:

`POST /api/v1/amocrm/oauth/exchange`

```json
{
  "code": "authorization_code_from_amocrm"
}
```

Важно:
- текущая реализация хранит токены в памяти процесса (MVP-уровень);
- после рестарта сервиса потребуется повторная авторизация;
- на следующем этапе вынесем токены в базу данных.
